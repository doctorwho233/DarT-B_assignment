### 11.5. 모델 성능 향상을 위한 파생 변수 생성

<!-- 새롭게 배운 내용을 자유롭게 정리해주세요. -->
개념

파생 변수(Feature Engineering)는 기존 데이터를 변환·조합하여 모델이 더 잘 학습하게 만드는 변수를 생성하는 작업이다.

주요 기법

산술 결합: 합, 곱, 평균, 비율 등

날짜 기반 파생: 월, 분기, 요일, 시점 차이(Δ)

로그 변환: 비대칭 분포를 정규화

범위 구분 변수: 구간화(binning)

핵심 포인트

파생 변수는 “데이터 이해 기반”으로 설계해야 의미가 있음

과적합을 유발할 수 있어 검증 데이터 기준으로 성능 평가 필요

모델링 성능의 50% 이상은 ‘데이터와 피처 엔지니어링’에서 결정됨
### 11.6. 슬라이딩 윈도우 데이터 가공

<!-- 새롭게 배운 내용을 자유롭게 정리해주세요. -->
개념

시간 기반 데이터에서 현재 시점 기준 과거 N개월의 값을 함께 구성해 주는 방식.
→ 예측 모델이 “고객의 최근 패턴”을 학습하도록 지원.

원리

기준 시점(예: 2022-05-01)을 기준으로

1개월 전, 2개월 전 … N개월 전 값을 새로운 컬럼으로 확장

이렇게 복제된 구조를 슬라이딩 윈도우 형태 테이블이라 부름

사용 목적

고객 구매 추세 파악

시계열 기반 예측모델(이탈, 재구매, 구매액 예측 등)에 활용

과거 값을 피처로 사용하기 위해 데이터 구조 전환

핵심 포인트

“시점 간 차이(MM_DIFF)” 변수는 매우 중요한 판단 지표

기준일을 무엇으로 잡느냐에 따라 파생변수 값도 달라짐

조인, 집계, 피벗 등 판다스 활용 능력이 핵심
### 11.7. 범주형 변수의 가변수 처리

<!-- 새롭게 배운 내용을 자유롭게 정리해주세요. -->
개념

문자형·범주형 변수는 모델이 직접 처리하지 못하므로
0/1로 변환한 새로운 컬럼(가변수)을 생성해야 한다.

방법

One-hot encoding (가장 일반적)

예: 성별 → Male(1/0)

첫 번째 범주 제거(drop_first=True)
다중공선성(multicollinearity) 방지 목적

결측값 옵션(dummy_na=True)
결측을 하나의 범주로 인정할 때 사용

주요 원칙

k개 범주 → k−1개 가변수 생성

baseline(제거된 첫 범주)에 대한 상대적 영향을 학습

범주 수가 너무 많을 경우는 “희소성(sparsity)” 문제 발생

주의점

과도한 가변수 생성 주의
(예: 제품명 740개 → 컬럼 740개 생성 → 차원의 저주)

모델 입력 전 꼭 차원 축소 여부 검토 필요
### 11.8. 클래스 불균형 문제 해결을 위한 언더샘플링과 오버샘플링

<!-- 새롭게 배운 내용을 자유롭게 정리해주세요. -->
문제 정의

타깃(y) 값의 비율이 극단적으로 치우친 상황.
예: 구매(1)=10%, 미구매(0)=90%

→ 모델은 “대부분 0이라고 예측”해도 정확도가 높아짐
→ 실제로는 정확도는 높지만 성능이 최악인 문제 발생

언더샘플링(Under-sampling)

큰 클래스 데이터 수를 줄여 작은 클래스와 균형을 맞추는 방식

종류

Random Under-sampling
큰 클래스에서 무작위 제거

EasyEnsemble
큰 클래스를 여러 번 분할해 여러 모델을 학습 → 결과 앙상블

CNN (Condensed Nearest Neighbor)
가장 결정에 필요한 샘플만 남기는 방식

장점

간단하고 빠름

큰 클래스의 과도한 영향 방지

단점

정보 손실 큼

데이터가 적을수록 쓰기 어려움

오버샘플링(Over-sampling)

작은 클래스의 데이터를 증가시키는 방식

종류

Random Over-sampling: 단순 복제

SMOTE(Synthetic Minority Over-sampling Technique)
소수 클래스 주변 데이터를 기반으로 “가상 데이터 생성”

ADASYN: 소수 클래스 중에서도 어려운 데이터에 더 많이 샘플 생성

장점

데이터 소실 없음

작은 클래스 학습 강화

단점

과적합 위험

생성 데이터 품질이 낮을 수 있음
### 11.9. 데이터 거리 측정 방법

<!-- 새롭게 배운 내용을 자유롭게 정리해주세요. -->
개념

데이터 간 유사도/차이를 수치화한 것.
머신러닝(특히 분류·군집)에 매우 중요.

1) 유클리드 거리(Euclidean)

가장 일반적인 거리. 두 점 사이의 직선거리.

기하학적 직관 있음

값이 작을수록 더 유사

2) 맨해튼 거리(Manhattan, cityblock)

격자 형태 경로의 거리

L1 Norm

절댓값의 합

3) 민코프스키 거리(Minkowski)

유클리드·맨해튼의 일반화

p=1 → 맨해튼

p=2 → 유클리드

p>2 → 더 높은 차수 거리

4) 체비쇼프 거리(Chebyshev)

두 점의 축 방향 차이 중 최댓값

격자 기반 이동에서 활용

max(|a−b|)

5) 마할라노비스 거리(Mahalanobis)

공분산을 고려한 거리

변수 간 상관관계 반영

분포 기반 측정

이상치 탐지에 유용

6) 코사인 거리(Cosine Distance)

두 벡터의 각도 기반 유사도

방향성이 중요할 때 사용

고차원 텍스트 데이터에서 특히 중요

코사인 유사도 = cosθ, 코사인 거리 = 1 - cosθ
